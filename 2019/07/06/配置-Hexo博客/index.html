前言

其实去年夏天就接触过博客了，只是搭建好之后就没有去深究它。前段时间忙面试的事情就发现身边的同学简历上都有博客，抱着好奇的心态去看看他们博客上都有哪些内容，结果发现他们将学习中的点点滴滴都记录在博客上了，我发现这也是一个不错的方式，可以供自己还有其他人去学习你的经历与领悟。于是，我决定也开始玩一玩博客了。于是，就有了第一篇文章，记录搭建博客的过程。

在搭建博客前，你需要完成以下内容…

1.安装Git
2.安装NodeJS
3.安装Hexo
4.生成本机SSH并添加到Github
5.部署项目
6.上传到github
7.修改及配置主题

开始啦

1.安装Git

你可以选择不安装Git，但是这样操作起来会比较麻烦，就是每次都要打开cmd…
我的电脑是widows环境，在这里就介绍windows版本的安装使用了

  • 下载地址
  • 安装步骤：双击exe文件，next安装
  • 安装好后，打开gitBash，查看版本（使用命令：git version）
  • 显示正确的版本后，你就安装好了

2.安装NodeJS

Hexo是基于NodeJS环境的静态博客，要使用里面的npm包，因此，安装NodeJS非常有必要

  • 下载地址
  • 安装步骤：打开msi文件，双击打开，next安装，在Custom Setup这一步选择 Add to PATH，这样就不用自己去配置环境变量了，装完打开cmd，输入path就可以看到你的node是否配置在里面了。
  • 查看版本，输入命令行 node -v
  • 显示正确的版本后，你就安装成功了

3.安装Hexo

Hexo只要在gitBash里面安装即可

  • 先创建一个存放blog的文件夹，然后 cd 到该文件夹
  • 安装hexo的命令：npm install -g hexo
  • 安装完成后，查看版本：hexo -v
  • 显示正确版本后，可以执行 hexo init 进行初始化操作
  • 初始化完成后，可以看到一系列文件
  • 文件解释如下：
    node_modules：依赖包
public：存放的是生成的页面
scaffolds：命令生成文章等的模板
source：用命令创建的各种文章
themes：主题
_config.yml：整个博客的配置
db.json：source解析所得到的
package.json：项目所需模块项目的配置信息
  • 接下来就是各种配置啦…

4.搭桥到github

创建账号什么的这里就不说了，按照流程走不会错…

  • 创建一个仓库，名称为yuorname.github.io，其中yourname是你的github的名称，温馨提示：只有按照这个规则创建才有效哦
  • 回到gitbash中，配置github账户信息，执行如下命令(将姓名和邮箱替换成自己的)：
    git config --global user.name "shiyisanyecao"
git config --global user.email "2327532812@qq.com"
  • 创建ssh，在gitbash中输入：
    ssh-keygen -t rsa -C "youremail@example.com
  • 生成ssh，然后找到 id_rsa.pub 中的内容，将ssh放到github中
  • 在gitbash中验证是否添加成功：
    ssh -T git@github.com

5.部署项目

  • 用编辑器打开blog项目，修改 _config.yml 文件里的一些配置
    deploy:
  type: git
  repo: https://github.com/YourgithubName/YourgithubName.github.io.git
  branch: master
  • 回到gitbash中，进入你的blog目录，分别执行以下命令
    hexo clean
hexo generate
hexo server
  • 打开浏览器输入：http://localhost:4000
  • 你就可以看到自己的blog了，是不是hen激动…

6.上传到github

  • 先安装一波：npm install hexo-deployer-git –save
  • 执行命令：
    hexo clean
hexo generate
hexo deploy
  • 在浏览器中输入：http://yourgithubname.github.io 就可以看到你的个人博客了
  • 有没有超兴奋

7.修改及配置主题

hexo初始化后默认的主题为 landscape ，你可以去这个地址找到你想要的主题。在github中搜索你要的主题名称，按照该主题的使用介绍进行改就好了。

8.写文章

  • 新建文章 hexo new ‘文章标题’ ，然后你就可以在 source/_posts 路径下看到你创建的文章了，编辑完成之后按照上面的方式部署
  • 可以查看官网
  • markdown文件也可以自己百度到…
