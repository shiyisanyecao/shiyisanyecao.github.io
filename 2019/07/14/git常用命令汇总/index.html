如果你是一个开发人员，想用上这个世界上目前最先进的分布式版本控制系统，那么，赶快开始学习吧！

Git操作

安装git后第一步

配置名字和邮箱，之后的每一步操作都需要这些（以我自己的为例）

git config --global user.name "shiyisanyecao"
git config --global user.email "2327532812@qq.com"

获取git配置信息

git config --list

创建版本库

创建版本库其实就是创建目录，此目录里的所有文件都可以被git管理起来，每个文件的修改、删除，git都能追踪，以便出现错误时可以还原

mkdir demo
cd demo
git init

创建好之后会有一个.git文件，不过该文件默认是隐藏的

把文件添加到版本库

git add .
git commit -m '说明'

仓库状态

git status

工作区和暂存区

工作区就是在电脑里能看到的目录，比如之前创建的demo就是工作区

把git添加到版本库是分两步执行的：

  • git add 把文件添加进去，事实上是添加到暂存区
  • git commit提交更改，实际上是把暂存区的内容提交到当前分支

删除文件

如果是把文件管理器里面的删了，直接执行下面命令：

rm readme.md
  • 如果你确定要从版本库中删除文件，则执行如下命令：
git rm readme.md
git commit -m "remove readme.md"

这样，你就从版本库中被删除了

  • 如果是你删错了，这时候版本库中还有，所以可以很轻松的把文件恢复
git checkout -- readme.md

生成SSH Key

创建SSH Key。在用户主目录下看看有木有.ssh目录，如果有，再看看这个目录下有没有id_rsa和id_rsa.pub文件。如果已经有了，则跳过这一步。如果没有，打开git Bash，创建SSH Key：

ssh-keygen -t rsa -C "2327532812@qq.com"

之后你就可以看到目录下多了两个文件id_rsa和id_rsa.pub。id_rsa是私钥，不能告诉别人。id_rsa.pub是公钥，可以放心的告诉别人。
然后登陆github，进入setting，SSH Keys页面，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容。
这样就ok了

远程服务器

git remote add origin git@github.com:shiyisanyecao/shiyisanyecao.github.io.git
git push -u origin master

从远程库克隆

git clone git@github.com:shiyisanyecao/shiyisanyecao.github.io.git
